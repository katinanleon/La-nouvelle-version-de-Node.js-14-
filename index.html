LE NOUVEAU Node.js v14

C'est quoi Node.js ?

NodeJS c’est une plateforme de devoppement qui met à disposition plusieurs bibliothèques JavaScript directement sur votre machine. Ce n’est pas un serveur, ni un framework, mais un applicatif permettant d’interagir par le biais du langage JavaScript et ainsi effectuer des actions en fonction des entrées / sorties détectées. NodeJS repose entièrement sur le moteur V8 pour fonctionner et permet d’avoir des performances de très haut niveau. D’ailleurs une des utilisations possibles de NodeJS consiste à s’en servir en tant que Serveur Web. Il permet de réaliser les mêmes actions que d’autres langages comme PHP ou Python et s’impose de plus en plus dans ce domaine. L’un des atouts majeurs de NodeJS est justement ce qui le caractérise, c’est-à-dire l’utilisation du JavaScript. JavaScript est un langage très simple d’apprentissage et très répandu du fait de son utilisation dans les navigateurs Web. Le fait d’utiliser plusieurs langages différents dans un projet peut rapidement mener à des problèmes de raisonnement ou d’écriture. En utilisant du JavaScript cotés navigateur, mais aussi côté serveur cela permet de simplifier les choses et de construire une solution rapidement. L’autre atout que possède NodeJS concerne cette fois-ci sa communauté qui est très active. En effet si vous souhaitez réaliser des opérations complexes, comme interroger une base de données ou automatiser certaines taches lourdes telles que la lecture de fichier, il est probable qu’un autre développeur ait créé ce qu’on appelle un « module » pour effectuer cette action et qu’il soit disponible pour vous.

L'évolution du Node.js

La base de Node.js

Comme on peut le voir Node.js est basé sur JavaScript qui est langage qui existe depuis longtemps, mais a connu une révolution ces dernières années notamment avec une technologie appelée le moteur V8. Cette technologie créée par Google a pour but d’améliorer de façon significative les performances lors de l’exécution d’instruction JavaScript. Cette technologie intégrée dans un premier temps au navigateur Google Chrome est par la suite à été adapté au sein d’autres navigateurs tel que Safari, Firefox, Opéra.
Notons que la version recente est Node.js 13

Node.js Version 14

La nouvelle Node.js est Node.js 14

Les responsables du projet Node.js ont fait savoir cette semaine la disponibilité de la version 14.0 de la célèbre plate-forme JavaScript. Ils soulignent toutefois que cette nouvelle version 14.0 n'est pas prête pour la production tant qu'elle n'atteindra pas le statut LTS, prévu pour octobre 2020. En attendant, la communauté des utilisateurs est encouragée à signaler les éventuels problèmes qui pourraient être rencontrés avec node.js 14.0. Dans Node.js 14.0 le rapport de diagnostic devient une fonctionnalité stable. Le rapport de diagnostic vous permet de générer un rapport à la demande ou lorsque certains événements se produisent. Ce rapport contient des informations qui peuvent être utiles pour diagnostiquer les problèmes de production, notamment les plantages, les performances lentes, les fuites mémoire, une charge élevée du processeur, les erreurs inattendues, etc. Dans Node.js 14.0, le moteur JavaScript V8 est mis à niveau vers la version 8.81. Celle-ci apporte de meilleures performances et de nouvelles fonctionnalités JavaScript, dont le chaînage optionnel et l'opérateur de coalescence null. A remarquer encore : Les packages écrits dans Web Assembly pour Node.js offrent la possibilité d'améliorer les performances et la prise en charge multi plates-formes pour certains cas d'utilisation. Cette nouvelle version node.js 14.0 inclut une implémentation expérimentale de l'interface du système d'assemblage Web (WASI) afin d'aider à prendre en charge ces cas d'utilisation. Enfin signalons que cette version 14.0 apporte une API de stockage local Async expérimentale, qui a également été rétroportée en 13.10. Le projet souhaite que la communauté essaie cette nouvelle API et donne ses commentaires sur le modèle d'abstraction, l'interface API, la couverture des cas d'utilisation, la stabilité fonctionnelle la documentation, etc.
